O que é

UML (Unified Modeling Language) é uma linguagem de modelagem padronizada utilizada na engenharia de software. Ela surgiu da necessidade de unificar as diversas notações e abordagens de modelagem que existiam na década de 1990. A história do UML pode ser dividida em algumas fases importantes:

1. Origens (década de 1980 - início de 1990):

Antes do UML, várias metodologias e notações de modelagem eram utilizadas na indústria de software.

Grady Booch desenvolveu uma metodologia de desenvolvimento de software que utilizava diagramas para visualizar o sistema, conhecida como "Booch Method".

James Rumbaugh desenvolveu a metodologia "Object Modeling Technique (OMT)".

Ivar Jacobson desenvolveu a metodologia "Object-Oriented Software Engineering (OOSE)".

2. Unificação (meados da década de 1990):

Em 1994, Grady Booch, James Rumbaugh e Ivar Jacobson uniram forças para criar uma linguagem de modelagem unificada que pudesse incorporar as melhores práticas de suas respectivas metodologias.

Em 1995, eles fundaram o consórcio "UML Partners" para desenvolver o UML como uma linguagem padrão para modelagem de software.

3. Lançamento do UML (1997):

Em 1997, a primeira versão do UML foi lançada. Essa versão inicial continha nove tipos de diagramas para representar diferentes aspectos de um sistema, como diagramas de classes, diagramas de objetos, diagramas de casos de uso, entre outros.

O UML foi bem recebido pela comunidade de desenvolvimento de software devido à sua capacidade de representar visualmente sistemas complexos.

4. Padronização (década de 2000 em diante):

O UML foi padronizado pela Object Management Group (OMG), uma organização sem fins lucrativos, que se tornou responsável por seu desenvolvimento contínuo.

O UML continuou a evoluir, com o lançamento de várias versões, cada uma trazendo melhorias e refinamentos na linguagem.

5. Uso Generalizado (até os dias atuais):

O UML tornou-se uma ferramenta padrão na indústria de software para modelar sistemas complexos.

É amplamente utilizado por analistas de sistemas, desenvolvedores e arquitetos de software para projetar e documentar sistemas de software.
